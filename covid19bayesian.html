A probabilistic model

 
xxxxxxxxxx
1

md"### A probabilistic model"
 9.8 μs

Let's think about how we would probablistically simulate the bottom figure given the top figure. We'll make several simplifying assumptions:

  1. Every infectious person is reported at the time they become infectious. This is an oversimplification and should be addressed to obtain a more complete model.

  2. Each person is infectious for a specific number of days, which we'll call γ⁻¹.

Each day, the number of new reported cases is obtained as a sum of the cases generated by each infectious person over the previous γ⁻¹ days. An infectious person infects γRₜ persons on average each day, so we can obtain the mean of the number of new cases by multiplying γRₜ for each of the previous γ⁻¹ days by the corresponding day's number of new infections, and totaling the results.

The mean of this random variable approximately specifies its distribution, because it's approximately binomially distributed with large n and small p. In other words, the random variable is approximately Poisson.

Let's write this up in code. If cases is the vector of case counts, we have

for i in 2:n
    cases[i] ~ Poisson(max(0, Rₜ[i] / γ⁻¹ * sum(cases[max(1, i-γ⁻¹):i-1])))
end

Now suppose we have the case values and want to infer the values of Rₜ. The first thing to observe is that the Rₜ values are not determined uniquely because of the randomness in the generation of the Poisson random variables. So the conditional distribution of Rₜ given cases is the best we can hope for.

Remarkably, we can solve this apparently very complex inverse problem by just handing it to Turing (!!). We supply the observed values and call Turing's sample function.
